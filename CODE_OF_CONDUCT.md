# Code of Conduct

## Our Standard

Participation in this project is expected to stay respectful, direct, and technical.

Examples of expected behavior:

- discuss code and behavior, not people
- assume good faith, but be explicit about risks and tradeoffs
- accept review feedback without hostility
- keep bug reports and pull requests actionable

Examples of unacceptable behavior:

- harassment, threats, or discriminatory language
- personal attacks or repeated bad-faith argument
- publishing private or security-sensitive information irresponsibly
- spam, trolling, or deliberately disruptive behavior

## Enforcement

Project maintainers may edit, hide, lock, or remove comments, issues, pull requests, or contributors that violate this code of conduct.

## Reporting

If conduct becomes a problem, report it through repository maintainers or the same private path used for security-sensitive reports when appropriate.
